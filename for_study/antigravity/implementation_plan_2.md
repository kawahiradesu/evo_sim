# Step 2: Perlin Noiseによる地形生成

## 目的

現在の「左半分がジャングル、右半分が砂漠、中央に直線の川」という単純な環境を、**Perlin Noiseで生成された自然な地形**に置き換えます。これにより「山の裏側は乾燥」「谷に水が集まって川ができる」「尾根で生物集団が隔離され種分化が起きる」といった、仕様書が目指す**「地形が進化圧の主要ドライバー」**を実現します。

## 背景知識：なぜPerlin Noiseなのか？

### ランダムノイズ vs Perlin Noise

| 方式 | 見た目 | 特徴 |
|---|---|---|
| `np.random.random()` | 砂嵐（全ピクセルがバラバラ） | 隣接セル間に連続性がない |
| **Perlin Noise** | なだらかな山脈・丘陵 | **隣接セル間がなめらかに繋がる** |

Perlin Noiseの核心は「**格子点にランダムな勾配ベクトルを置き、その間を滑らかに補間する**」ことです。これにより、自然界の地形のような「近いところは似た値、遠いところは違う値」というパターンが生まれます。

### なぜ外部ライブラリを使わないのか？

`pip install noise` や `opensimplex` を使えば一行で済みますが、以下の理由で**自作**します：
1. **学習目的**: Perlin Noiseの仕組みを理解することがこのプロジェクトの目的
2. **依存関係ゼロ**: NumPyだけで完結するので環境構築のトラブルがない
3. **カスタマイズ性**: 地形生成パラメータを自由に調整できる

## Proposed Changes

### 地形生成ユーティリティ

#### [NEW] [terrain.py](file:///Users/yooseiisikawa/sakuhin/evo_sim/src/terrain.py)

Perlin Noiseの実装と、それを使った地形グリッド生成を行うモジュール。

**実装する関数と、各関数が「なぜ必要か」の解説：**

##### 1. `perlin_noise_2d(shape, scale, octaves, persistence, seed)`
```
【何をする関数？】
  2Dのパーリンノイズマップ（0.0〜1.0の連続値を持つ2D配列）を生成する。

【なぜこの設計？】
  - scale: 地形の「スケール」を制御。小さいと細かい丘、大きいと巨大な山脈
  - octaves: ノイズを複数重ねて「大きな山 + 小さな丘 + 微細な凸凹」を作る
    （これを「フラクタルノイズ」と呼ぶ。現実の地形も同じ性質を持つ）
  - persistence: 重ねるごとに振幅をどれだけ小さくするか（0.5が標準）
  - seed: 再現性のためのシード値（同じseedなら同じ地形が生成される）
```

##### 2. `generate_altitude(shape, seed)` → `altitude_grids`
```
【何をする関数？】
  高度マップを生成する。0.0（海面/最低地）〜 1.0（山頂）

【なぜ必要？】
  仕様書Part 4で「高度・地形生成が最優先」と書かれている。
  高度がmoisture（水分）やriver（川）の源泉になるため、
  他のすべてのグリッドの「親」にあたる。
```

##### 3. `generate_river(altitude_grids)` → `river_grids`
```
【何をする関数？】
  高度マップから「水は高いところから低いところに流れる」原理で
  川の位置を決定する。高度が低い谷筋に川ができる。

【なぜ必要？】
  Step 1で作ったriver_gridsを「直線の川」から「地形に沿った自然な川」に
  進化させる。engine.pyは一切変更不要（データ駆動設計の恩恵！）
```

##### 4. `generate_moisture(altitude_grids, river_grids)` → `moisture_grids`
```
【何をする関数？】
  高度と川の位置から水分量を計算する。
  ルール:
    - 川の近くは水分が高い（農耕地帯のイメージ）
    - 高い山の上は乾燥している（風が水分を奪うため）
    - 低地は水分が溜まりやすい

【なぜ必要？】
  現在の「左=1.0, 右=0.0」という不自然な分断を、
  地形に連動した自然なグラデーションに置き換える。
  これにより「川沿いにジャングルが生え、山頂は草原」という
  バイオームの自然な空間分布が生まれる。
```

## Open Questions

> [!IMPORTANT]
> **Q1: 地形の毎回ランダム生成 vs 固定シード**
> `TERRAIN_SEED = 42` で固定すると毎回同じ地形になります。デバッグ中は同じ地形のほうが結果を比較しやすいですが、完成後はランダムにしたいですか？ → とりあえず固定シードで実装し、後からランダムに切り替えられるようにします。

> [!NOTE]
> **Q2: 高度による移動速度への影響**
> 山を登ると遅くなる（移動コスト増加）という仕様を **このステップで入れるか、後回しにするか**？
> → 提案: 地形生成と描画を先に完成させ、移動への影響はStep 2.5として分離します。
