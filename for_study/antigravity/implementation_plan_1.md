# 今後の開発方針（開発ロードマップ提案）

`SPEC_NEWed.MD` の「Part 4: 未解決・今後の設計課題」および全体の設計思想に基づき、今後の開発をスムーズかつ論理的に進めるためのフェーズ別ロードマップを提案します。

環境（土台）の構築から始め、徐々に個体や生態系全体への影響へ広げていくアプローチを採用しています。

## User Review Required

この開発方針（ロードマップ）の優先順位やアプローチについて、ご意見・修正点があればお知らせください。特に「Phase 1」から着手して問題ないか、あるいは別の機能（Emergent Traitなど）を優先したいかご確認ください。

## フェーズ別開発ロードマップ

### Phase 1: 進化圧の土台となる環境・地形基盤の完成
最も優先度が高い「高度・地形生成」と「エネルギー保存則」を実装し、シミュレーションの物理的な土台を固めます。

#### 1. 高度・地形生成 (`altitude_grids`) の実装 [優先度: 高]
- **目的**: 進化圧の主要ドライバーとなる地理的な多様性（山脈、盆地など）を生み出す。
- **実装内容**: Perlin Noise または Simplex Noise を用いて、初期化時に連続的な高度マップを生成。高度が温度や移動コストに影響を与えるようにする。

#### 2. エネルギー保存則の完全実装 [優先度: 高]
- **目的**: 「無から有が生じない」閉じたエコシステムを完成させる。
- **実装内容**: 太陽光（Layer 0からの定数入力）を植物（Layer 2）の成長エネルギー源として定義。現在の植物成長ロジックを、太陽光エネルギーの吸収と土壌/水分の消費に基づくモデルに改修する。

---

### Phase 2: 創発的形質 (Emergent Traits) と局所環境の深化
個体が環境に適応するための非線形な能力（派生値）と、地形に基づく環境の動的変化を実装します。

#### 1. Emergent Trait（派生値）の計算ロジック実装 [優先度: 高]
- **目的**: 複数の遺伝子の掛け算による「派生能力（飛行、高知能など）」を創発させる。
- **実装内容**: `engine.py` の個体更新プロセスに、基本遺伝子群から派生値（例: `flight_score = wing * metabolism * balance`）を計算する関数を追加し、移動速度や捕食回避率などの実際のパラメーターに反映させる。

#### 2. 河川システム (`river_grids`) の導入 [優先度: 中]
- **目的**: 水分供給の偏りを作り、オアシスや乾燥地帯といった局所的なバイオーム形成を促進する。
- **実装内容**: Phase 1で作った高度情報に基づき、高いところから低いところへ流れる水流アルゴリズムを実装し、周辺の `moisture_grids` に影響を与えるようにする。

---

### Phase 3: マクロな生態系制御と世界規模の変動
増えすぎた個体群の自然な間引きや、長期的な環境変動（Layer 0）を実装します。

#### 1. 密度依存疫病 (`pathogen_grids`) の実装 [優先度: 中]
- **目的**: MAX_TARO（個体数上限）到達時の不自然な挙動を解消し、自然な個体群制御を行う。
- **実装内容**: 個体が密集している領域で病原菌濃度が上昇するグリッドを追加。個体の免疫（腸内細菌 `t_microbiome` などと連動）を判定し、感染ダメージを与える。

#### 2. Layer 0 (Planetary Layer) の進行とイベント実装 [優先度: 中]
- **目的**: 季節変動や気候変動による長期的な進化圧の付与。
- **実装内容**: 
  - `climate_phase` の導入により、定期的な `moisture_grids`（乾季・雨季）や温度の変動を実装。
  - ランダムな World Event（隕石、植物革命など）の確率発火メカニズムを追加。

---

### Phase 4: システムの堅牢化・最適化
動作安定性やメンテナンス性を向上させます。

#### 1. Spawn Queue のリファクタリング [優先度: 低]
- **実装内容**: 現在 `attempt_mating` 内で直接配列を書き換えている処理を、一旦キュー（Spawn Queue）に溜め、Update Pipelineの最後（ステップ6）で一括適用する設計にリファクタリングし、同フレーム内の依存バグを防止する。

#### 2. 時間スケール（`t_ages`）の調整 [優先度: 低]
- **実装内容**: 年齢や成熟にかかるフレーム数と、現実時間（UIでの見え方）の対応付けを整理・調整する。

## Verification Plan

各フェーズの実装後、以下の方法で動作を検証します。
1. **可視化とテレメトリ**: 導入したグリッド（高度、河川、病原菌など）をUI上で切り替えて可視化できるようにし、意図通り分布しているか目視確認。
2. **ログ出力**: エネルギーの総量が系全体で保存されているか、毎秒の総エネルギー量を計算してコンソールにアサートする。
3. **長期シミュレーション**: 10万tick以上回し、MAX_TAROのクラッシュが起きないか（疫病等による制御）、地形や派生形質による種分化が実際に起きるかを観測。
