## Perlin Noiseの仕組み（図解）

### 🧮 STEP 1: 格子点に勾配ベクトルを配置

```
   ↗ ─────── ↙        各格子点（交差点）に
   │         │        ランダムな方向の矢印を置く。
   │    ・P   │        この矢印が「地形の傾き」を決める。
   │         │
   ← ─────── ↑
```

### 🧮 STEP 2: 内積（ドット積）で値を計算

ピクセルPの値 = 周囲4つの格子点の「勾配ベクトル」と「格子点→Pベクトル」の内積

- 勾配の方向とPへの方向が **同じ** → 正の値（高い）
- 勾配の方向とPへの方向が **逆** → 負の値（低い）

### 🧮 STEP 3: スムーズ補間でブレンド

4つの値を`_fade()`で滑らかにした重みで混ぜる。
→ 格子の境界が見えない、なめらかなノイズが完成！

### 🎵 フラクタルノイズ（オクターブの重ね合わせ）

```
大きな山脈（周波数=1, 振幅=1.0）
    ＋
中くらいの丘（周波数=2, 振幅=0.5）
    ＋
小さな起伏（周波数=4, 振幅=0.25）
    ＋
微細な凸凹（周波数=8, 振幅=0.125）
    ＝
自然な地形！
```

## 因果の連鎖（データ駆動設計の真骨頂）

```
                      ┌──────────────┐
                      │ Perlin Noise │
                      └──────┬───────┘
                             ↓
                     ┌───────────────┐
                     │ altitude_grids│ ← 高度（すべての親）
                     └───────┬───────┘
                  ┌──────────┴──────────┐
                  ↓                     ↓
          ┌──────────────┐    ┌──────────────┐
          │ river_grids  │    │    高度補正    │
          │  （低地=川）  │    │ （高地=乾燥） │
          └──────┬───────┘    └───────┬──────┘
                 │                    │
                 └────────┬───────────┘
                          ↓
                 ┌──────────────────┐
                 │ moisture_grids   │ ← 川＋高度から計算
                 └────────┬─────────┘
                          ↓
                 ┌──────────────────┐
                 │ process_plants() │ ← moisture参照で成長
                 │ （engine.py）     │
                 └────────┬─────────┘
                          ↓
                 ┌──────────────────┐
                 │ 動物の生存・進化  │
                 └──────────────────┘
```

> [!TIP]
> **データ駆動設計の恩恵**: Step 1で`river_grids`をグリッド参照にしたおかげで、**engine.pyは今回1行も変更していない**。地形データを差し替えるだけで、直線の川が自然に蛇行する川に変わった。

## 学べるポイントまとめ

| # | 概念 | 具体的にどこで使ったか |
|---|---|---|
| 1 | **Perlin Noise** | `perlin_noise_2d()` — 勾配ベクトル＋内積＋スムーズ補間 |
| 2 | **フラクタルノイズ** | `octaves`パラメータ — 異なるスケールのノイズを重ねて自然な地形に |
| 3 | **ベクトル化** | `np.meshgrid` — forループの代わりにNumPy配列操作で全ピクセルを一括処理 |
| 4 | **因果の連鎖** | `altitude→river→moisture` — 1つのデータが次のデータを決定する設計 |
| 5 | **データ駆動設計** | engine.pyの変更ゼロで地形が完全に変わった |
| 6 | **色の線形補間** | `lerpColor()` — フロントエンドでRGB各成分を個別にブレンド |
